## 🎯 阶段1：配置GitHub Secrets和Variables（必须）

### 步骤1.1：进入配置页面

1. 打开你的GitHub仓库页面
2. 点击 **Settings** 标签
3. 左侧菜单选择 **Secrets and variables** → **Actions**
4. 你会看到两个标签页：**Secrets** 和 **Variables**

---

### 步骤1.2：配置基础股票池（Secrets）

**为什么放在Secrets**：基础股票池包含你的实盘持仓信息，属于敏感数据。

1. 点击 **Repository secrets** 标签页
2. 点击 **New repository secret** 按钮
3. 填写：
   ```
   Name:  BASE_STOCK_LIST
   Value: 600519,000001  # 替换为你的实盘持仓股票代码
   ```
4. 点击 **Add secret** 保存

**说明**：
- 多个股票代码用逗号分隔，不要有空格
- 如果不配置此项，将完全依赖自动筛选（无fallback）
- 建议配置2-5只你长期持有的股票

---

### 步骤1.3：配置自动筛选开关（Variables）

1. 点击 **Repository variables** 标签页
2. 点击 **New repository variable** 按钮
3. 填写：
   ```
   Name:  AUTO_SCREEN_ENABLED
   Value: true
   ```
4. 点击 **Add variable** 保存

**说明**：
- 设为 `true` 启用自动筛选
- 设为 `false` 或不配置则使用传统手动模式

---

### 步骤1.4：配置筛选参数（Variables，可选）

继续添加以下变量（使用默认值可跳过）：

#### 调整最大收盘价

```
Name:  SCREENER_MAX_PRICE
Value: 50  # 默认50元，可调整范围30-100
```

**说明**：只筛选收盘价低于此价格的股票

#### 调整筛选数量

```
Name:  SCREENER_TOP_N
Value: 5  # 默认5只，可调整范围3-10
```

**说明**：按成交额排序，返回前N只股票

---

### 步骤1.5：验证配置

配置完成后，你的页面应该显示：

**Repository secrets**:
- `BASE_STOCK_LIST` ✅

**Repository variables**:
- `AUTO_SCREEN_ENABLED` = `true` ✅
- `SCREENER_MAX_PRICE` = `50` (可选) ✅
- `SCREENER_TOP_N` = `5` (可选) ✅

## ⚠️ 故障排查指南

### 问题1：筛选器未执行

**现象**：日志中没有"股票筛选"相关输出

**可能原因**：
- `AUTO_SCREEN_ENABLED` 未设为 `true`
- 工作流未正确触发

**解决**：
1. 检查Variables中 `AUTO_SCREEN_ENABLED` 是否为 `true`
2. 手动触发一次工作流验证

---

### 问题2：筛选失败

**现象**：日志显示"筛选失败"或"FALLBACK到基础池"

**可能原因**：
- 网络问题（无法访问东方财富服务器）
- 数据源暂时不可用
- 市场休市（周末/节假日）

**解决**：
- 等待下一个交易日重试
- 检查GitHub Actions日志中的具体错误信息
- 如果是网络问题，通常会自动重试

---

### 问题3：筛选结果为空

**现象**：没有股票符合筛选条件

**可能原因**：
- 价格条件太严格（如 `SCREENER_MAX_PRICE` 太低）
- 市场整体上涨，低价股很少

**解决**：
- 放宽价格限制（提高 `SCREENER_MAX_PRICE`）
- 检查基础池是否有股票

---

### 问题4：基础池股票未出现

**现象**：最终列表中没有基础池股票

**可能原因**：
- 基础池股票与自动筛选结果重复（正常去重）
- `BASE_STOCK_LIST` 配置错误

**解决**：
1. 检查Secrets中 `BASE_STOCK_LIST` 是否正确配置
2. 查看日志中的"基础股票池已配置"输出
3. 如果重复，这是正常行为（优先使用自动筛选的详细信息）

---

### 问题5：GitHub Actions运行失败

**现象**：工作流显示红色❌

**可能原因**：
- 依赖安装失败
- 网络超时
- 配置错误

**解决**：
1. 查看完整的运行日志
2. 找到具体的错误信息
3. 根据错误提示修复

**常见错误**：
- `ModuleNotFoundError`: 依赖缺失 → 检查requirements.txt
- `ConnectTimeout`: 网络超时 → 通常会自动重试
- `ConfigError`: 配置错误 → 检查Secrets/Variables
