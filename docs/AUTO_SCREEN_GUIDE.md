# 🤖 自动股票筛选快速入门

> 每个交易日自动筛选股票，与手动维护的基础股票池合并，实现"自动筛选 + 实盘持仓"的混合模式。

## 📋 功能特性

- ✅ **自动筛选**：每个交易日根据预设策略自动筛选股票
- ✅ **基础股票池**：手动维护实盘持仓或长期关注股票
- ✅ **智能合并**：自动筛选结果与基础池合并（去重）
- ✅ **Fallback 机制**：筛选失败时自动降级到基础池
- ✅ **灵活配置**：支持自定义筛选参数

## 🎯 筛选策略（当前实现）

| 条件 | 说明 |
|------|------|
| 市场范围 | 沪深主板（剔除创业板 300xxx、科创板 688xxx） |
| 风险剔除 | 剔除 ST 股票 |
| 价格约束 | 收盘价 < 50 元（可配置） |
| 排序规则 | 按成交额降序排列 |
| 返回数量 | 前 N 只（可配置，默认 5 只） |

## ⚡ 快速配置（3 步开始）

### 步骤 1：配置基础股票池（可选但推荐）

在 GitHub 仓库的 **Settings → Secrets and variables → Actions → Secrets** 中添加：

```
Name: BASE_STOCK_LIST
Value: 600519,000001,300750  # 你的实盘持仓或长期关注股票
```

### 步骤 2：启用自动筛选

在 **Variables** 中添加：

```
Name: AUTO_SCREEN_ENABLED
Value: true
```

### 步骤 3：调整筛选参数（可选）

在 **Variables** 中添加：

```
Name: SCREENER_MAX_PRICE
Value: 50  # 最大收盘价（元）

Name: SCREENER_TOP_N
Value: 5   # 返回股票数量
```

### 场景 1：筛选失败（数据源不可用）

```

```

**结果**：使用基础池的 2 只股票

## 🎛️ 配置模式对比

| 模式 | AUTO_SCREEN_ENABLED | BASE_STOCK_LIST | STOCK_LIST | 效果 |
|------|---------------------|-----------------|------------|------|
| **完全自动** | `true` | 不配置 | 自动填充 | 每天筛选 5 只新股票 |
| **混合模式**（推荐） | `true` | `600519,000001` | 自动填充 | 5 只自动 + 基础池 |
| **手动配置** | `false` | 不配置 | `600519,000001` | 仅分析手动配置的股票 |
| **仅基础池** | `false` | `600519,000001` | 不配置 | 仅分析基础池股票 |

## ⚠️ 注意事项

1. **交易时间**：筛选器在每个交易日 18:00（北京时间）执行，使用当天的收盘数据
2. **数据源**：依赖 AkShare 获取 A 股实时行情，确保 GitHub Actions 网络可达
3. **基础池维护**：基础股票池需要手动更新（通过 GitHub Secrets）
4. **去重逻辑**：自动筛选的股票如果在基础池中存在，会优先使用自动筛选的结果
5. **性能影响**：筛选过程约 5-10 秒，对整体分析时间影响很小
6. **首次使用**：建议先本地测试，确认筛选结果符合预期后再启用

## 🔧 故障排查

### 问题 1：筛选器报错"No module named 'akshare'"

**解决**：确保 `requirements.txt` 已正确安装

```bash
pip install -r requirements.txt
```

### 问题 2：筛选结果为空

**可能原因**：
- 市场休市（周末或节假日）
- 价格条件太严格（如 `SCREENER_MAX_PRICE` 设得太低）
- 数据源暂时不可用

**解决**：
- 检查是否为交易日
- 放宽价格条件（如提高到 60 元）
- 查看日志确认具体原因

### 问题 3：基础池股票未出现在最终列表

**可能原因**：基础池股票与自动筛选结果重复

**解决**：检查筛选结果，重复的股票会使用自动筛选的详细信息（包含收盘价、成交额等）

### 问题 4：GitHub Actions 执行失败

**检查清单**：
- [ ] `BASE_STOCK_LIST` 是否在 Secrets 中正确配置
- [ ] `AUTO_SCREEN_ENABLED` 是否在 Variables 中设为 `true`
- [ ] 查看 Actions 日志，确认筛选器输出
- [ ] 检查是否有网络限制或数据源访问问题

## 🚀 进阶用法

### 场景 1：根据市场热度调整筛选数量

```yaml
# 市场活跃时多筛选一些
SCREENER_TOP_N: "10"

# 市场冷清时少筛选一些
SCREENER_TOP_N: "3"
```

### 场景 2：根据资金量调整价格范围

```yaml
# 资金量小，关注低价股
SCREENER_MAX_PRICE: "30"

# 资金量大，可以关注中价股
SCREENER_MAX_PRICE: "80"
```

### 场景 3：季节性调整

```yaml
# 年报季关注业绩股，放宽价格限制
SCREENER_MAX_PRICE: "100"
SCREENER_TOP_N: "10"

# 平时保持保守策略
SCREENER_MAX_PRICE: "50"
SCREENER_TOP_N: "5"
```
